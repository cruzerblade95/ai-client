# Security Policy

## Supported versions

During the pre-1.0 development period, security fixes
will be applied to the latest published release.

| Version        | Supported |
| -------------- | --------- |
| Latest release | Yes       |
| Older releases | No        |

## Reporting a vulnerability

Do not disclose security vulnerabilities in public
GitHub issues, discussions, or pull requests.

Use GitHub's private vulnerability reporting feature
for this repository when available.

Include:

- A description of the vulnerability
- The affected version
- Reproduction steps
- The potential impact
- Any suggested mitigation

Do not include real AWS credentials, npm tokens,
personal data, or other secrets in the report.

Please allow reasonable time for investigation before
public disclosure.
